# mouse_modes (AutoHotkey v2)

Небольшой скрипт для Windows, который добавляет “режимы” для мыши:
- временное изменение **системной скорости курсора** при удержании клавиши
- режим **скролла движением мыши** (Raw Input): курсор фиксируется в точке, а движение мыши превращается в прокрутку

> Требования: AutoHotkey **v2**.

---

## Как это работает

### Скорость мыши
При удержании клавиши скрипт:
1) запоминает текущую системную скорость (1..20)
2) выставляет скорость медленнее/быстрее (относительно базовой)
3) при отпускании возвращает исходную скорость

### Скролл движением мыши
При удержании клавиши скрипт:
1) фиксирует курсор в текущей точке (ClipCursor)
2) читает движение мыши через **Raw Input (WM_INPUT)**
3) отправляет события колёсика (`WheelUp/Down/Left/Right`)
4) при отпускании отпускает курсор

---

## Хоткеи

Все режимы работают **только пока клавиша зажата**.

- **F13** — медленная мышь  
- **F14** — быстрая мышь  
- **F15** — горизонтальный скролл (влево/вправо)  
- **F16** — вертикальный скролл (вверх/вниз)  
- **F15 + F16** — скролл по обеим осям одновременно

---

## Настройка

Вверху `mouse.ahk` можно менять:

- `speedStep` — на сколько менять системную скорость (1..20)
- `rawUnitsPerNotch` — чувствительность скролла (меньше = быстрее)
- `maxNotchesPerMsg` — лимит “щелчков” колеса за одно сообщение Raw Input

---

## Запуск

### Через AutoHotkey
Дважды кликни `mouse.ahk` (или запусти через AutoHotkey v2).

### Компиляция в exe
Можно скомпилировать скрипт стандартными средствами AutoHotkey v2 и запускать как обычную программу.

---

## Примечания

- В некоторых приложениях/играх (особенно с Raw Input) поведение может отличаться.
- Скрипт старается корректно освобождать `ClipCursor` и возвращать скорость при завершении.
